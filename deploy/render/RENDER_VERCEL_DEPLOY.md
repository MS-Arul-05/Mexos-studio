# Deploy — Backend on Render, Frontend on Vercel

Target architecture:

```
Browser ──► Vercel (Next.js, repo: mexos-studio-frontend)
                │  NEXT_PUBLIC_API_URL
                ▼
         Render (Express API, repo: Mexos-studio) ──► Neon Postgres (ap-southeast-1)
                                                 └──► S3 ap-south-1 (design uploads)
```

The apps live in **separate repos**: this repo is the backend; the frontend was
split out to `mexos-studio-frontend` (2026-07-09, history preserved).

Prerequisites: both repos pushed to GitHub, a Render account, a Vercel account.
Both platforms deploy on push once connected.

---

## 1. Backend → Render

The blueprint at the repo root (`render.yaml`) defines everything: Node 22 web
service in **Singapore** (next to Neon), build = install + prisma generate +
tsc + `prisma migrate deploy`, health check on `/api/health/live`.

1. Render Dashboard → **New → Blueprint** → select the GitHub repo → Apply.
2. Render prompts for the `sync: false` env vars. Fill them:

   | Var             | Value                                                      |
   | --------------- | ---------------------------------------------------------- |
   | `DATABASE_URL`  | Neon **pooled** connection string (keep `sslmode=require`) |
   | `CORS_ORIGINS`  | placeholder for now — set after Vercel gives you the URL   |
   | `S3_ACCESS_KEY` | IAM key for the uploads bucket                             |
   | `S3_SECRET_KEY` | IAM secret                                                 |

   JWT secrets and `ADMIN_API_KEY` are auto-generated by Render (strong, 32+
   chars) — the production boot audit validates them and refuses placeholders.

3. First deploy runs migrations automatically. Verify:

   ```
   curl https://mexos-api.onrender.com/api/health        # {"db":"up"}
   curl https://mexos-api.onrender.com/api/v1/products?limit=1
   ```

Boot-audit expectations on this initial config (all intentional):

- `OTP_PROVIDER=console` → **warning** only; OTPs appear in Render logs
  (Dashboard → Logs, search `console-otp`). Switch to WhatsApp per
  `GOING_LIVE.md` §2 once the Business API is approved.
- Payments unset → **warning**, stub gateway. Add `PAYMENT_GATEWAY_KEY_ID`,
  `PAYMENT_GATEWAY_KEY_SECRET`, `PAYMENT_WEBHOOK_SECRET` together — setting
  only one is a fatal boot error by design.

## 2. Frontend → Vercel

1. Vercel Dashboard → **Add New → Project** → import the
   **`mexos-studio-frontend`** repo (its root is the Next.js app; leave Root
   Directory and build/output at their defaults — Next.js auto-detects).
2. Environment variable (all environments):

   ```
   NEXT_PUBLIC_API_URL=https://mexos-api.onrender.com/api/v1
   ```

   (Use your real Render service URL; add `/api/v1`.)

3. Deploy, note the production URL (e.g. `https://mexos-studio.vercel.app`).

## 3. Wire them together

1. Back in Render → mexos-api → Environment → set:

   ```
   CORS_ORIGINS=https://mexos-studio.vercel.app
   ```

   Comma-separate additional origins (custom domain) as they appear. Vercel
   _preview_ deployments have unique URLs that are not in this allowlist —
   their server-rendered pages work, but browser-side API calls are blocked
   unless you add the preview URL. Unset/empty `CORS_ORIGINS` is a fatal boot
   error in production (the API refuses to allow all origins).

2. Add the Vercel origin to the **S3 bucket CORS** config (S3 console →
   bucket → Permissions → CORS), so browser-direct presigned uploads work.
   Currently only `http://localhost:3000` is allowed:

   ```json
   [
     {
       "AllowedOrigins": ["http://localhost:3000", "https://mexos-studio.vercel.app"],
       "AllowedMethods": ["PUT"],
       "AllowedHeaders": ["Content-Type", "Content-Length"],
       "MaxAgeSeconds": 3600
     }
   ]
   ```

3. Save → Render redeploys → smoke test in the browser:
   - `/shop` lists the catalog (client-side fetch → proves CORS)
   - a product page loads (SSR fetch → proves server-to-server)
   - login sends an OTP → code visible in Render logs while on `console`

## 4. Razorpay webhook (when live keys arrive)

Dashboard → Settings → Webhooks → `https://mexos-api.onrender.com/api/payments/webhook`,
subscribe `payment.captured` + `payment.failed`, and set the same secret in
`PAYMENT_WEBHOOK_SECRET`.

## Operational notes

- **Free-plan cold starts**: Render free instances spin down after 15 min idle
  (~50 s to wake) and Neon autosuspends too. For a live shop, use the Starter
  plan and consider disabling Neon autosuspend.
- **Migrations** run in the build step because `preDeployCommand` needs a paid
  plan. On Starter+, move `npx prisma migrate deploy` from `buildCommand` to
  `preDeployCommand` in `render.yaml`.
- **Uploads** go straight from the browser to S3 via presigned URLs — no
  Render disk involved, nothing extra to configure.
- Going-live switches for OTP/WhatsApp/payments: see `GOING_LIVE.md`.
